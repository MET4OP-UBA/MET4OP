Segn el teorema de la estructura, cualquier programa de computadora puede escribirse utilizando las estructuras de control bsicas . Una estructura de control (o flujo de control) es un bloque de programacin que analiza variables y elige una direccin en la que ir en funcin de parmetros dados. En pocas palabras, una estructura de control es solo una decisin que toma la computadora. Por lo tanto, es el proceso bsico de toma de decisiones en programacin y el flujo de control determina cmo responder un programa de computadora cuando se le den ciertas condiciones y parmetros.

Hay dos aspectos bsicos de la programacin de computadoras: datos e instrucciones . Para trabajar con datos, se debe comprender las variables y los tipos de datos (clase_01); para trabajar con instrucciones, necesita comprender las estructuras y declaraciones de control (clase_02). El flujo de control a travs de cualquier programa dado se implementa con tres tipos bsicos de estructuras de control: secuencial, seleccin y repeticin

El teorema del programa estructurado es un resultado en la teora de lenguajes de programacin. Establece que toda funcin computable puede ser implementada en un lenguaje de programacin que combine slo tres estructuras lgicas. Esas tres formas (tambin llamadas estructuras de control) especficamente son:

Secuencial¶

La ejecucin secuencial es cuando las declaraciones se ejecutan una tras otra en orden. No necesitas hacer nada ms para que esto suceda.

Selección¶

Ejecucin de una de dos instrucciones (o conjuntos), segn el valor de una variable booleana. Utilizada para decisiones, ramificacin: elegir entre 2 o ms caminos alternativos.

  • if
  • if ... else
  • switch

Repetición¶

ejecucin de una instruccin (o conjunto) mientras una variable booleana sea 'verdadera'. La repeticin se utiliza para hacer bucles, es decir, repetir un fragmento de cdigo varias veces seguidas.

  • while loop
  • do..while loop
  • for loop

Estas estructuras de control se pueden combinar en programacin informtica. Una secuencia puede contener varios bucles; un bucle puede contener un bucle anidado dentro de l, o las dos ramas de un condicional pueden contener cada una secuencias con bucles y ms condicionales. Contadores¶

Un contador es una variable de tipo numrico destinada a contener diferentes valores, que se van incrementando o disminuyendo en cantidades fijas. En otras palabras un contador es una variable que actualiza su valor en trminos de su propio valor anterior y de una constante. Puede utilizarse, por ejemplo, para contar la cantidad de puntos que realiza un equipo.

Los contadores se utilizan con la finalidad de contar sucesos o acciones internas de un ciclo, deben realizar una operacin de inicializacin y posteriormente las sucesivas de incremento o decremento del mismo.

La inicializacin consiste en asignarle al contador un valor. Se sita antes y fuera del ciclo.

Descripcin General

Nombre_del_Contador = Nombre_del_Contador + Valor_Constante

Donde:

  • Nombre_del_Contador es un nombre para la variable contador.
  • Valor_Constante la constante de incremento. Puede ser 1 u otro valor.

Ejemplo VOTOS = VOTOS + 1 Cada vez que ingresa un voto, la cantidad de votos aumenta en 1.

Acumuladores¶

Es una variable que suma sobre si misma un conjunto de valores para de esta manera tener la suma de todos ellos en una sola variable. En otras palabras, un acumulador es una variable que actualiza su valor en trminos de su propio valor anterior y el valor de otra variable u otra expresin.

Descripcin General

Nombre_del_Acumulador = Nombre_del_Acumulador + Valor_Variable

Donde:

  • Nombre_del_Acumulador: es un nombre de identificador vlido.
  • Valor_Variable: variable de tipo numrico (entero o real).

Ejemplo TOTAL = TOTAL + SUBTOTAL

In [ ]:
#Ejemplo en Python
acumulador = acumulador + variable
La diferencia entre un contador y un acumulador es que mientras el primero va aumentando de forma constante, el acumulador va aumentando en cantidad variable.

Inicialización de las variables¶

  • Si deseamos sumar o restar, debemos inicializar la variable en 0. De esta forma nos aseguramos de que no existan valores residuales.
  • Si deseamos multiplicar o dividir, debemos inicializar la variable. Caso contrario obtendremos siempre 0.

La instruccin pass es una declaracin nula. La diferencia entre un comentario y una declaracin pass en Python es que mientras el intrprete ignora un comentario por completo, el pass no es ignorado.

Generalmente lo usamos como placeholder.

Supongamos que tenemos un ciclo o una funcin que an no est implementada, pero queremos implementarla en el futuro. No pueden tener un cuerpo vaco. El intrprete dara un error. Entonces, usamos la instruccin pass para construir un cuerpo que no hace nada por el momento.
